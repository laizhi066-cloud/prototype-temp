# 原型评审

像产品经理评审原型一样检查。

## 检查清单

- 产品目标和目标用户是否能从流程中看出来。
- 项目类型和采用的默认骨架是否匹配。
- 后台页面是否包含登录、头部、侧边栏、内容区和 Element Plus 组件。
- C 端和 H5 页面是否没有误用后台侧边栏。
- 核心流程是否可以点击走通。
- 加载、空状态、错误、校验、成功反馈和危险操作确认是否完整。
- 布局和组件是否复用，而不是每个页面重写一套。
- 数据是否通过 `src/mock/mockApi.js` 流转。
- 桌面和移动端视口下文字是否溢出或遮挡。
- 是否避免了巨型单文件输出和无关框架改动。

## 完成规则

声明完成前必须运行构建。涉及视觉或交互时，启动本地应用并使用浏览器或截图检查。
