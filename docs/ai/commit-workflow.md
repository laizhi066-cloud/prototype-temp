# AI 提交回溯规则

产品同事不需要理解 Git，但每轮 AI 修改都应该留下一个可回溯节点。

## 给 AI 的硬性要求

- 每次完成有效文件修改后，必须创建 Git commit。
- commit 信息必须使用中文。
- 不要使用纯英文提交信息，例如 `update files`、`fix bug`。
- 最终回复必须告诉产品同事本轮 commit 信息。
- 如果无法提交，必须明确说明原因，不能假装已经提交。

## 推荐 commit 信息

```text
初始化产品原型脚手架
完善 PM 使用入口和提示词库
优化登录页展示
新增订单管理模块
调整后台主题和字号体系
修复移动端导航体验
```

## AI 可使用的命令

```bash
npm run commit:ai -- "完善 PM 使用入口和提示词库"
```

这个命令会：

1. 检查当前目录是否是 Git 仓库；
2. 暂存本轮修改；
3. 使用中文信息创建 commit；
4. 如果没有可提交内容，会给出提示。

## PM 怎么要求 AI

```text
本轮修改完成后，请运行验证，并创建一个中文 commit。
最终回复告诉我 commit 信息。
```
