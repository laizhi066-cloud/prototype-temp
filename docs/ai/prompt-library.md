# PM 常用提示词库

复制下面的提示词，把方括号内容替换成你的需求即可。

## 新建一个后台管理系统

```text
请先读取 AGENTS.md，并使用 product-prototype-orchestrator。

我想做一个 [系统名称]，主要给 [目标用户] 使用。
核心业务是 [核心流程]。
必须包含 [关键模块或页面]。

请先判断项目类型并说明置信度。
如果判断为后台管理 / SaaS / CRM / 运营系统，请默认包含登录页、固定头部、树状侧边栏、内容区、用户菜单和退出登录。
请先推荐主题、配色、布局密度和登录页方向。
如果信息不够，每轮最多问 1 个关键问题。
确认默认假设后直接实现可运行原型。
完成后请运行验证，并创建一个中文 commit。
```

## 新建一个 CRM / 客户管理系统

```text
请使用 product-prototype-orchestrator、project-intake-classifier 和 product-theme-advisor。

我要做一个 CRM 客户管理系统。
目标用户是销售主管和销售人员。
核心流程包括客户录入、客户分级、跟进记录、商机阶段、待办提醒。

请按 SaaS / CRM 默认骨架处理。
不要问是否需要登录、侧边栏和头部。
请推荐适合销售团队日常高频使用的主题和信息密度，然后实现第一版原型。
完成后请创建中文 commit。
```

## 新建一个 H5 移动端页面

```text
请使用 product-prototype-orchestrator。

我想做一个手机端 [页面 / 活动 / 流程]。
用户是 [目标用户]。
用户要完成 [核心动作]。

请按 H5 移动端处理，默认单列布局、触控友好表单、顶部返回或底部 Tab。
不要生成后台侧边栏。
如果信息不够，只问我一个最影响流程的问题。
完成后请创建中文 commit。
```

## 新建一个 C 端 Web 产品

```text
请使用 product-prototype-orchestrator 和 product-theme-advisor。

我想做一个 C 端 Web 产品：[产品名称]。
用户是 [用户群体]。
核心路径是 [浏览 / 搜索 / 预约 / 下单 / 个人中心]。

请默认使用顶部导航、首页、列表页、详情页和个人中心。
不要套后台管理侧边栏。
请先推荐主题、视觉风格和页面结构，再实现可点击原型。
完成后请创建中文 commit。
```

## 新建一个数据看板

```text
请使用 product-prototype-orchestrator。

我要做一个 [业务名称] 数据看板。
使用者是 [管理者 / 运营 / 财务 / 销售]。
核心指标包括 [指标 1]、[指标 2]、[指标 3]。
需要支持按 [时间 / 区域 / 渠道 / 负责人] 筛选。

请默认包含顶部筛选区、指标卡、图表区、明细表格、加载态、空状态和错误态。
请推荐适合数据密集场景的主题和图表表达。
完成后请创建中文 commit。
```

## 新增一个业务模块

```text
请使用 feature-module-template。

在当前原型中新增 [模块名称] 模块。
目标用户是 [目标用户]。
核心流程是 [新增 / 编辑 / 删除 / 查询 / 审批 / 导出]。
主要字段有 [字段列表]。

请补齐页面、路由、树状侧边栏入口、mock API、本地存储、列表、筛选、分页、表单校验、成功反馈和危险操作确认。
完成后请创建中文 commit。
```

## 修改产品名称、主题和登录页描述

```text
请使用 product-context、brand-config 和 product-theme-advisor。

把当前原型改成 [产品名称]。
目标用户是 [目标用户]。
一句话描述是 [系统描述]。
品牌标识用 [1 个中文或英文缩写]。
主题希望体现 [专业 / 活泼 / 高端 / 效率 / 数据感]。

请统一修改产品名、登录页描述、侧边栏品牌、默认主题色和首页文案。
运行界面不要出现“原型”“脚手架”“AI”等产品外身份词。
完成后请创建中文 commit。
```

## 让 AI 先反问，但不要连环追问

```text
请使用 product-question-gate。

我会给你一个不完整的产品想法。
你先根据已有信息推断项目类型、用户、核心流程和默认页面。
如果必须提问，每轮只问 1 个最关键问题。
不要问登录页、头部、侧边栏、技术栈、组件库这些项目已经默认的问题。

我的想法是：[产品想法]
```

## 审查当前原型像不像真实产品

```text
请使用 ui-ux-pro-max 和 prototype-review。

请审查当前原型，重点看：
1. 是否像真实业务产品；
2. 是否有 AI 模板感；
3. 登录页是否只展示 logo、名称、描述和登录表单；
4. 侧边栏是否支持树状结构；
5. 字号是否以 12px 为紧凑基准但仍可读；
6. 移动端是否没有遮挡和横向滚动。

发现问题后直接优化，并用截图验证。
完成后请创建中文 commit。
```

## 生成 PRD

```text
请使用 prd-development。

请基于当前原型生成 PRD。
需要包含：背景、目标用户、问题、目标、范围、页面清单、核心流程、字段说明、状态规则、非目标、风险和验收标准。
文档使用 UTF-8 中文。
```

## 生成用户故事和验收标准

```text
请使用 user-story。

请把当前 [模块 / 页面 / 流程] 转成用户故事。
每条用户故事使用“作为 / 我想要 / 以便于”格式。
每条都补 Gherkin 验收标准。
```
