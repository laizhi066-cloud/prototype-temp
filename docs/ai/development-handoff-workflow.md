# 开发交付工作流

PM 确认原型后，下一步不是让开发“自己看页面理解”，而是让 AI 生成开发交付包。

## 一句话提示词

```text
请使用 prd-development、user-story、user-story-mapping 和 development-handoff。
基于当前原型生成交付开发的完整需求包。
需要包含 PRD、用户故事、字段 / 状态说明、接口 / mock 数据说明、验收用例、开发任务拆分、风险和未决问题。
文档放到 docs/product/ 对应目录，使用 UTF-8 中文。
完成后创建中文 commit。
```

## 交付包应该包含

- PRD：说明为什么做、给谁用、做什么、不做什么、怎么验收。
- 用户故事：把需求拆成开发可拿的故事和验收标准。
- 开发交接说明：页面、路由、字段、状态、接口、数据、权限和边界情况。
- 验收用例：PM、测试和开发都能按用例检查。

## 不合格交付

这些情况不能交给开发：

- 只有原型链接，没有 PRD。
- 只有页面截图，没有字段和状态说明。
- 只有功能列表，没有用户故事和验收标准。
- 只有 mock 数据，没有接口含义和数据结构说明。
- 没有写清非范围、风险和未决问题。

## PM 验收交付包

检查下面 6 点：

1. 开发能否知道第一版做什么、不做什么？
2. 每个页面是否有用途、入口、主操作和状态说明？
3. 每个表单字段是否有类型、是否必填、校验规则？
4. 每个关键流程是否有正常、失败、空数据和异常场景？
5. 是否有用户故事和可测试的验收标准？
6. 是否有开发任务拆分和建议优先级？
