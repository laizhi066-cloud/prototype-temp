# 反问式需求澄清模板

这份文档解决一个问题：PM 的需求经常不完整，但 AI 不能连续追问一堆配置问题。

## PM 直接复制版

```text
请使用 product-question-gate 和 project-intake-classifier。

我会描述一个产品想法，你先做合理推断，不要马上连续提问。

请按这个顺序输出：
1. 你推断的项目类型和置信度；
2. 你推断的目标用户；
3. 你推断的核心流程；
4. 你建议的默认页面骨架；
5. 你会采用的主题和布局密度；
6. 如果必须提问，只问 1 个最关键问题。

禁止问这些问题：
- 是否需要登录；
- 是否需要头部；
- 是否需要侧边栏；
- 是否使用 Vue / React / Tailwind；
- 是否使用 Element Plus；
- 是否需要 mock 数据。

我的需求是：[产品想法]
```

## AI 应该怎么问

### 情况 1：能判断为后台管理

用户说：

```text
做一个订单管理系统。
```

AI 应该回答：

```text
我先按后台管理处理，因为你提到了订单管理。
默认包含登录页、固定头部、树状侧边栏、订单列表、筛选、详情、新增、编辑、删除、分页和异常状态。
我会采用克制型 B 端运营后台主题。
如果你没有其他限制，我直接开始规划并实现第一版原型。
```

### 情况 2：能判断为 H5

用户说：

```text
做一个手机端活动报名。
```

AI 应该回答：

```text
我先按 H5 移动端处理，因为你提到了手机端和活动报名。
默认使用单列布局、顶部活动信息、报名表单、提交成功页和触控友好控件。
我不会使用后台侧边栏。
```

### 情况 3：无法判断项目类型

用户说：

```text
做一个会员系统。
```

AI 最多问：

```text
这是后台管理、C 端 Web，还是 H5？
```

不要继续追问登录、侧边栏、颜色、表格、字段等细节。

## 高影响问题库

只有这些问题值得问，且每轮只问 1 个：

- 这是后台管理、C 端 Web，还是 H5？
- 目标用户是谁？
- 核心流程是查询管理，还是用户自助操作？
- 这个页面主要用于转化报名，还是日常操作？
- 数据是以列表为主，还是以图表看板为主？

## 低价值问题库

这些问题不要问，直接按项目默认值处理：

- 要不要登录页？
- 要不要固定头部？
- 要不要侧边栏？
- 用 Vue 还是 React？
- 用不用 Element Plus？
- 要不要加载态、空状态、错误态？
- 要不要本地存储？
- 按钮要什么颜色？
