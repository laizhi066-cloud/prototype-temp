# 产品经理从这里开始

这份文档给不懂开发、不懂技术的产品同事使用。你不需要理解 Vue、Node.js、Vite，只需要会复制提示词、运行一键脚本、看页面是否符合需求。

## 第一次使用

在 Windows 电脑上，按顺序双击这些文件：

1. `scripts/check-for-pm.bat`
2. 如果提示缺少依赖，再双击 `scripts/setup-for-pm.bat`
3. 最后双击 `scripts/start-for-pm.bat`

启动成功后，终端会显示本地地址，通常是：

```text
http://127.0.0.1:5173/
```

默认登录信息：

- 账号：`ops.admin`
- 密码：`demo123`

如果电脑没有 Node.js，不要自己排查环境，把终端提示发给 AI，让 AI 按 `environment-bootstrap` 处理。

## 第一次让 AI 做原型

优先复制这段：

```text
请先读取 AGENTS.md，并使用 product-prototype-orchestrator、project-intake-classifier、product-question-gate 和 product-theme-advisor。

我想做一个：[一句话产品想法]
目标用户是：[谁使用]
核心场景是：[用户要完成什么]

请你先判断这是后台管理、C 端 Web、H5、活动页、数据看板还是工具型页面。
请先根据行业、用户、设备和信息密度推荐主题、配色、布局密度和登录页方向。
如果信息不够，只问我一个最关键的问题。
不要问我是否需要登录页、头部、侧边栏；后台类产品按项目默认规则处理。
确认方向后，直接生成可运行原型，并补齐加载、空状态、错误状态、表单校验和成功反馈。
```

更多现成提示词见：

- `docs/ai/prompt-library.md`
- `docs/ai/clarifying-question-prompts.md`
- `docs/ai/product-request-template.md`
- `docs/ai/commit-workflow.md`

## AI 应该怎么反问

AI 不应该一上来问一串：

- 要不要登录？
- 要不要顶部栏？
- 要不要侧边栏？
- 要什么颜色？
- 有哪些页面？

正确方式是：

1. 先判断项目类型。
2. 先给合理默认方案。
3. 只问一个会影响整体方向的问题。

例如你说「做一个会员系统」，AI 如果无法判断，最多只应该问：

```text
这是后台管理、C 端 Web，还是 H5？
```

## 怎么判断 AI 输出能不能接受

写代码前，AI 至少要说清楚：

- 项目类型和置信度。
- 推荐主题和不采用的风格。
- 默认页面和路由。
- 核心流程。
- 需要的 mock 数据。
- 验收标准。

如果 AI 跳过这些，直接生成一大段 HTML 或单文件代码，让它重新按本项目规则执行。

## 每轮修改都要有 commit

产品同事不需要自己操作 Git，但要要求 AI 每轮完成后创建中文 commit，方便回溯。

你可以直接说：

```text
本轮修改完成后，请运行必要验证，并创建一个中文 commit。
最终回复告诉我 commit 信息。
```

如果 AI 没有给出 commit 信息，不要认为这一轮已经完成回溯。

## 做完以后怎么验收

打开 `docs/ai/prototype-acceptance-checklist.md`，按清单检查。

最重要的 6 个问题：

1. 页面里是否出现真实业务名称，而不是“原型”“脚手架”“AI”？
2. 核心流程能不能点击走通？
3. 后台类产品是否有登录、头部、树状侧边栏和内容区？
4. 表单是否有校验，提交后是否有成功或错误反馈？
5. 没有数据、加载失败、删除等情况是否有处理？
6. 手机尺寸下文字是否溢出、按钮是否容易点击？

## 常见任务怎么说

新增模块：

```text
使用 feature-module-template。
我要新增一个 [模块名称] 模块，核心流程是 [流程]。
请补页面、路由、mock API、本地存储、加载态、空状态、错误态、表单校验和验收清单。
```

优化页面：

```text
使用 ui-ux-pro-max 和 prototype-review。
请审查当前 [页面名称]，重点看是否像真实业务产品、是否有 AI 模板感、字号是否过大、移动端是否可用。
发现问题后直接优化，并截图验证。
```

生成 PRD：

```text
使用 prd-development。
请基于当前原型生成一份 PRD，包含背景、用户、目标、范围、核心流程、页面说明、字段说明、状态规则和验收标准。
```
