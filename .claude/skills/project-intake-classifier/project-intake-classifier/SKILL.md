# 项目类型识别

先识别，再提问。根据名词、目标用户、设备线索和流程形态判断项目类型。

## 类型判断

- **后台管理 / SaaS / CRM / 运营系统：** 管理、订单、用户、商家、财务、审批、库存、工单、客服、内部、角色、权限、员工、数据看板。
- **C 端 Web：** 用户端、预约、购物、课程、内容、账号、个人中心、搜索、详情、结算。
- **H5 移动端：** 手机端、微信、活动报名、轻表单、扫码、触屏、底部 Tab。
- **活动页：** 落地页、活动、促销、海报、转化、报名、邀请。
- **数据看板：** KPI、指标、BI、报表、监控、趋势、图表。
- **工具型页面：** 计算器、生成器、转换器、上传器、单任务工具。

## 默认骨架

- 后台管理 / SaaS / CRM / 运营系统：登录页、固定头部、侧边栏、内容区、用户菜单、退出登录、列表、详情、新增、编辑、删除、筛选、分页。
- C 端 Web：顶部导航、首页、列表页、详情页、个人中心，默认不使用后台侧边栏。
- H5 移动端：移动端页面壳、顶部返回栏或底部 Tab、单列内容、卡片列表、触控友好表单。
- 活动页：聚焦一个转化流程，包含清晰行动按钮、表单或操作步骤、成功状态。
- 数据看板：筛选区、指标卡、图表区域、明细表格、导出占位。
- 工具型页面：输入区、操作按钮、结果区，必要时包含历史记录或示例状态。

## 输出要求

实现前必须用中文说明：

1. 推断出的项目类型；
2. 置信度：高 / 中 / 低；
3. 将采用的默认骨架；
4. 页面和路由；
5. 复用组件；
6. mock API 与 localStorage 数据；
7. 如果置信度低，只问 1 个关键问题。

如果置信度为中或高，直接按默认假设继续，并记录假设。
