# 产品追问闸门

少问，先判断。产品经理不应该回答实现层面的初始化琐事。

## 提问前检查

提问前先判断：

1. 答案能不能从需求中推断？
2. `AGENTS.md` 是否已经定义默认值？
3. 这个答案是否会实质改变原型？
4. 能否先带着明确假设继续？

只有当问题会实质改变原型，且不能合理假设时，才允许提问。

## 禁止问题

不要问：

- 后台管理、SaaS、CRM、运营系统是否需要登录页；
- 后台产品是否需要固定头部；
- 后台产品是否需要侧边栏；
- 本脚手架是否使用 Element Plus；
- 本脚手架是否使用 localStorage；
- 一轮内提出多个初始化问题。

## 默认问题

如果无法判断项目类型，只问：

> 这是后台管理、C 端 Web，还是 H5？

用户回答后，直接套用对应默认骨架。

## 回答风格

带假设继续时，用中文说明：

- 「我先按后台管理处理，因为你提到了订单 / 用户 / 审批。」
- 「我先按 H5 移动端处理，因为你提到了活动报名 / 手机端。」
- 「我先按 C 端 Web 处理，因为这是用户预约 / 浏览 / 下单流程。」
