# 产品上下文

运行界面必须像一个真实产品，而不是脚手架说明页。

## 核心规则

- 具体产品内容统一放在 `src/config/productProfile.js`。
- 脚手架说明只放在 README、docs 和 AGENTS.md，不出现在登录页、头部、侧边栏或业务页面。
- AI 接到真实需求后，先更新产品上下文，再新增页面或模块。
- 即使当前只是默认样板，运行界面也要呈现为真实业务系统；「示例」「脚手架」「原型」等说明只放在文档中。

## 必改字段

接到真实产品需求时，检查并更新：

- `productName`：登录页主标题。
- `shortName`：侧边栏品牌名称。
- `brandMark`：侧边栏品牌缩写。
- `subtitle`：侧边栏副标题。
- `projectType`：后台管理、C 端 Web、H5 移动端等。
- `targetUser`：目标用户。
- `loginEyebrow`：登录页辅助标题。
- `loginIntro`：登录页产品说明。
- `themeRecommendation`：根据行业、用户和信息密度形成的主题建议。
- `workspaceTitle`：后台首页标题。
- `workspaceSubtitle`：后台首页说明。
- `headerHint`：顶部栏当前系统提示。
- `navTree`：侧边栏树状导航，支持一级菜单和二级子菜单。
- `dashboardMetrics`：首页指标名称和值。

## 禁止事项

- 不要在运行界面展示「AI 原型脚手架」「产品原型脚手架」「Product Prototype Scaffold」等脚手架身份。
- 不要在登录页、侧边栏、顶部栏、首页和业务页面展示「示例」「原型」「AI」等产品外说明。
- 不要只改页面标题而不改侧边栏、登录页和首页指标。
- 不要把产品名散写到多个页面里。
- 不要在布局组件里写死导航菜单；后台、SaaS 和 CRM 默认使用 `productProfile.navTree` 生成树状侧边栏。
- 不要让 README 的脚手架说明污染运行界面。

## 验收

- 登录页显示具体产品名。
- 侧边栏显示具体产品名或简称。
- 侧边栏按 `navTree` 渲染，后台类产品默认支持树状导航。
- 首页标题和指标匹配业务场景。
- 搜索运行界面源码时，不应在 `src/` 中出现脚手架身份文案。
